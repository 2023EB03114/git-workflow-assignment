=== GIT COMMANDS USED IN ASSIGNMENT ===

1. git init
   Initializes a new empty Git repository in the current folder, creating a hidden .git directory.

2. git branch -M main
   Renames the current branch to 'main' (sets it as the default primary branch).

3. git status
   Displays the current state of the working directory and staging area, showing tracked/untracked files.

4. git add .
   Stages all new and modified files in the current directory for the next commit.

5. git add <filename>
   Stages a specific file for the next commit.

6. git commit -m "message"
   Records a snapshot of the staged changes to the repository with a descriptive message.

7. git branch feature
   Creates a new branch named 'feature' without switching to it.

8. git checkout feature
   Switches the working directory to the 'feature' branch.

9. git checkout main
   Switches back to the 'main' branch.

10. git merge feature -m "message"
    Merges the 'feature' branch into the currently active branch (main), combining their histories.

11. git log --oneline --graph --all
    Displays a compact, visual graph of the entire commit history across all branches.

12. git revert HEAD --no-edit
    Creates a new commit that undoes the changes from the most recent commit, preserving full history.

13. git remote add origin <url>
    Connects the local repository to a remote GitHub repository using the alias 'origin'.

14. git push -u origin main
    Pushes the local 'main' branch to GitHub and sets 'origin/main' as the upstream tracking branch.

15. git remote -v
    Lists all configured remote connections along with their fetch and push URLs.
